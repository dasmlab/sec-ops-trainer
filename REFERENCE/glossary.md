# Glossary - Key Terms and Acronyms

## Overview

This glossary provides definitions of key terms and acronyms used throughout the SecOps Standards Officer competency development plan.

---

## A

**ABAC** - Attribute-Based Access Control. Access control model based on attributes of users, resources, and environment.

**ACS** - Assertion Consumer Service. SAML endpoint where service providers receive authentication assertions.

**API** - Application Programming Interface.

**ARM TrustZone** - ARM's security extension providing a secure execution environment.

---

## B

**BGP** - Border Gateway Protocol. Exterior routing protocol used on the Internet.

**BIOS** - Basic Input/Output System. Firmware for system initialization (legacy, replaced by UEFI).

---

## C

**CA** - Certificate Authority. Entity that issues digital certificates.

**CaaS** - Container-as-a-Service. Cloud service model providing container orchestration platforms.

**CIA Triad** - Confidentiality, Integrity, Availability. Core principles of information security.

**CIDR** - Classless Inter-Domain Routing. IP addressing notation (e.g., 192.168.1.0/24).

**CIS** - Center for Internet Security. Organization that publishes security benchmarks (CIS Benchmarks).

**CISSP** - Certified Information Systems Security Professional. Advanced security certification from ISC².

**CKAD** - Certified Kubernetes Application Developer. CNCF certification for Kubernetes application developers.

**CKA** - Certified Kubernetes Administrator. CNCF certification for Kubernetes administrators.

**CKS** - Certified Kubernetes Security Specialist. CNCF certification for Kubernetes security.

**CNCF** - Cloud Native Computing Foundation. Organization that hosts Kubernetes and other cloud-native projects.

**CVE** - Common Vulnerabilities and Exposures. Standardized identifier for security vulnerabilities (format: CVE-YYYY-NNNNN).

**CVSS** - Common Vulnerability Scoring System. Framework for scoring vulnerability severity (Base, Temporal, Environmental scores).

**CWE** - Common Weakness Enumeration. List of software weaknesses.

---

## D

**DAC** - Discretionary Access Control. Access control model where resource owners control access.

**DDoS** - Distributed Denial of Service. Attack flooding a target with traffic from multiple sources.

**DISA** - Defense Information Systems Agency. U.S. DoD agency providing IT support. Publisher of STIGs.

**DMZ** - Demilitarized Zone. Network segment between internal and external networks.

**DNSSEC** - DNS Security Extensions. Security extensions for DNS providing origin authentication and integrity.

**DoH** - DNS over HTTPS. DNS resolution over HTTPS protocol.

**DSCP** - Differentiated Services Code Point. IP packet field for QoS.

---

## E

**EIGRP** - Enhanced Interior Gateway Routing Protocol. Cisco proprietary routing protocol.

**ESXi** - VMware hypervisor (Type 1).

**EVM** - Extended Verification Module. Linux kernel module for integrity measurement.

---

## F

**FAIR** - Factor Analysis of Information Risk. Quantitative risk analysis framework.

**FHS** - Filesystem Hierarchy Standard. Linux directory structure standard.

**FIPS** - Federal Information Processing Standard. U.S. government standards for computer systems.

**FTP** - File Transfer Protocol. Network protocol for file transfer.

**FTPS** - FTP over SSL/TLS. Secure FTP.

**FW** - Firewall. Network security device controlling traffic.

---

## G

**GDPR** - General Data Protection Regulation. EU regulation for data protection and privacy.

**GIAC** - Global Information Assurance Certification. Security certifications from SANS.

**GKE** - Google Kubernetes Engine. Google's managed Kubernetes service.

---

## H

**HDLC** - High-Level Data Link Control. Data link protocol.

**HIPAA** - Health Insurance Portability and Accountability Act. U.S. healthcare industry regulation.

**HSM** - Hardware Security Module. Hardware device for secure key storage and cryptographic operations.

**HTTP** - Hypertext Transfer Protocol. Application protocol for web.

**HTTPS** - HTTP over SSL/TLS. Secure HTTP.

---

## I

**IaaS** - Infrastructure-as-a-Service. Cloud service model providing virtualized infrastructure.

**ICMP** - Internet Control Message Protocol. Network layer protocol for error reporting.

**IdP** - Identity Provider. Entity that authenticates users and provides identity assertions (SAML/OIDC).

**IDS** - Intrusion Detection System. Security system monitoring for suspicious activity.

**IEEE** - Institute of Electrical and Electronics Engineers. Standards organization (e.g., 802.11 for wireless).

**IETF** - Internet Engineering Task Force. Standards organization (RFCs).

**IMAP** - Internet Message Access Protocol. Email protocol.

**IPS** - Intrusion Prevention System. Security system that actively blocks threats.

**IPSec** - IP Security. Protocol suite for secure IP communications.

**ISC²** - International Information System Security Certification Consortium. Security certification body (CISSP, etc.).

**ISMS** - Information Security Management System. Framework for managing security (ISO 27001).

**ISO** - International Organization for Standardization. International standards body (e.g., ISO 27001).

**ITSG** - Information Technology Security Guidance. Canadian government security guidance (ITSG-33, ITSG-38, etc.).

**ITU** - International Telecommunication Union. UN agency for ICT standards.

---

## J

**JIT** - Just-In-Time. Access model providing temporary access when needed.

**JSON** - JavaScript Object Notation. Data format (used in OIDC).

---

## K

**KVM** - Kernel-based Virtual Machine. Linux virtualization technology (hypervisor).

**K8s** - Kubernetes. Container orchestration platform (abbreviation).

---

## L

**LDAP** - Lightweight Directory Access Protocol. Protocol for directory services.

**LUKS** - Linux Unified Key Setup. Disk encryption specification for Linux.

**LVM** - Logical Volume Manager. Storage management for Linux.

---

## M

**MAC** - Mandatory Access Control. Access control model where access is controlled by system policy.

**MFA** - Multi-Factor Authentication. Authentication using multiple factors.

**MPLS** - Multi-Protocol Label Switching. WAN technology for packet forwarding.

**MTU** - Maximum Transmission Unit. Maximum packet size.

---

## N

**NAC** - Network Access Control. Network security controlling device access.

**NAT** - Network Address Translation. Translating IP addresses.

**NIST** - National Institute of Standards and Technology. U.S. standards organization (NIST CSF, NIST RMF, etc.).

**NVD** - National Vulnerability Database. NIST database of CVEs with additional data (CVSS scores).

---

## O

**OCSP** - Online Certificate Status Protocol. Protocol for checking certificate revocation status.

**OIDC** - OpenID Connect. Authentication protocol built on OAuth 2.0.

**OSI** - Open Systems Interconnection. Seven-layer network model.

**OSPF** - Open Shortest Path First. Link-state routing protocol.

**OWASP** - Open Web Application Security Project. Organization focused on application security (OWASP Top 10).

---

## P

**PaaS** - Platform-as-a-Service. Cloud service model providing application platforms.

**PAM** - Privileged Access Management. Management of privileged accounts and access.

**PAT** - Port Address Translation. NAT variant translating ports.

**PDP** - Policy Decision Point. XACML component making access decisions.

**PEP** - Policy Enforcement Point. XACML component enforcing access decisions.

**PHI** - Protected Health Information. Healthcare information protected by HIPAA.

**PIP** - Policy Information Point. XACML component providing attribute information.

**PKCE** - Proof Key for Code Exchange. OAuth 2.0 extension for public clients.

**PKI** - Public Key Infrastructure. Framework for digital certificates and public key cryptography.

**POP3** - Post Office Protocol version 3. Email protocol.

**PPP** - Point-to-Point Protocol. Data link protocol.

**PV** - Physical Volume. LVM storage component.

---

## Q

**QoS** - Quality of Service. Network traffic prioritization.

---

## R

**RBAC** - Role-Based Access Control. Access control model based on user roles.

**RDP** - Remote Desktop Protocol. Microsoft protocol for remote desktop access.

**RFC** - Request for Comments. IETF standards documents.

**RHEL** - Red Hat Enterprise Linux. Enterprise Linux distribution.

**RHCA** - Red Hat Certified Architect. Advanced Red Hat certification (requires 5 certifications).

**RHCE** - Red Hat Certified Engineer. Intermediate Red Hat certification.

**RHCSE** - Red Hat Certified Specialist in Security: Linux. Red Hat security specialization.

**RHCSA** - Red Hat Certified System Administrator. Foundational Red Hat certification.

**RIP** - Routing Information Protocol. Distance-vector routing protocol.

**RMF** - Risk Management Framework. NIST framework for managing security and privacy risks (SP 800-37).

**RP** - Relying Party. OIDC/OAuth client application.

**RPO** - Recovery Point Objective. Maximum acceptable data loss in disaster recovery.

**RTO** - Recovery Time Objective. Maximum acceptable downtime in disaster recovery.

---

## S

**S2I** - Source-to-Image. Build process for container images from source code.

**SAML** - Security Assertion Markup Language. XML-based protocol for identity federation (SAML 2.0).

**SaaS** - Software-as-a-Service. Cloud service model providing complete applications.

**SCAP** - Security Content Automation Protocol. Standardized format for security checklists and scanning.

**SD-WAN** - Software-Defined Wide Area Network. WAN technology using software-defined networking.

**SEV** - Secure Encrypted Virtualization. AMD technology for VM memory encryption.

**SGX** - Software Guard Extensions. Intel technology for secure enclaves.

**SIEM** - Security Information and Event Management. Security systems for log aggregation and analysis.

**SLO** - Single Logout. SAML feature for logging out of multiple services.

**SMTP** - Simple Mail Transfer Protocol. Email protocol.

**SNMP** - Simple Network Management Protocol. Network management protocol.

**SP** - Service Provider. SAML entity providing services to users.

**SPF** - Sender Policy Framework. Email authentication method.

**SRG** - Security Requirements Guide. DISA technology-agnostic security requirements (basis for STIGs).

**SSH** - Secure Shell. Protocol for secure remote access.

**SSO** - Single Sign-On. Authentication method allowing access to multiple services with one login.

**SSSD** - System Security Services Daemon. Linux service for identity management integration.

**STIG** - Security Technical Implementation Guide. DISA security configuration guides for technologies.

**STP** - Spanning Tree Protocol. Network protocol preventing loops in switched networks.

---

## T

**TACACS+** - Terminal Access Controller Access-Control System Plus. Authentication protocol.

**TCP** - Transmission Control Protocol. Connection-oriented transport protocol.

**TEE** - Trusted Execution Environment. Secure execution environment.

**TLS** - Transport Layer Security. Cryptographic protocol (successor to SSL).

**TOTP** - Time-based One-Time Password. MFA method (RFC 6238).

**TPM** - Trusted Platform Module. Hardware security module for key storage and attestation.

---

## U

**UDP** - User Datagram Protocol. Connectionless transport protocol.

**UEFI** - Unified Extensible Firmware Interface. Firmware interface (successor to BIOS).

**USB** - Universal Serial Bus. Interface standard.

---

## V

**VLAN** - Virtual Local Area Network. Logical network segmentation.

**VLSM** - Variable Length Subnet Masking. Subnetting with different subnet mask sizes.

**VM** - Virtual Machine. Virtualized computing environment.

**VPC** - Virtual Private Cloud. Cloud networking construct.

**VPN** - Virtual Private Network. Secure network connection over public networks.

**VXLAN** - Virtual Extensible LAN. Network virtualization technology.

---

## W

**WAF** - Web Application Firewall. Firewall protecting web applications.

**WAN** - Wide Area Network. Network spanning large geographical areas.

**WPA** - Wi-Fi Protected Access. Wireless security standard (WPA, WPA2, WPA3).

---

## X

**XACML** - eXtensible Access Control Markup Language. XML-based language for access control policies.

**XML** - eXtensible Markup Language. Markup language (used in SAML, XACML).

---

## Z

**ZTNA** - Zero Trust Network Access. Zero Trust approach to network access.

**ZTA** - Zero Trust Architecture. Security model with "never trust, always verify" principle.

---

## Standards Bodies and Organizations

**BSI** - British Standards Institution. Standards organization.

**CIS** - Center for Internet Security. Security organization (CIS Benchmarks).

**CNCF** - Cloud Native Computing Foundation. Cloud-native projects organization.

**DISA** - Defense Information Systems Agency. U.S. DoD IT support agency (STIGs).

**IANA** - Internet Assigned Numbers Authority. IP address allocation authority.

**IEEE** - Institute of Electrical and Electronics Engineers. Standards organization.

**IETF** - Internet Engineering Task Force. Internet standards organization (RFCs).

**ISO** - International Organization for Standardization. International standards body.

**ITU** - International Telecommunication Union. UN ICT standards agency.

**NIST** - National Institute of Standards and Technology. U.S. standards organization.

**OWASP** - Open Web Application Security Project. Application security organization.

---

## Additional Terms

**Authentication** - Verifying identity (who you are).

**Authorization** - Granting permissions (what you can do).

**Defense in Depth** - Security strategy using multiple layers of controls.

**Least Privilege** - Principle of granting minimum necessary permissions.

**Privilege Escalation** - Gaining higher-level permissions than authorized.

**Root of Trust** - Foundation of security chain (hardware or software).

**Zero Trust** - Security model with "never trust, always verify" principle.
